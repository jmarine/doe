  • Latest Release: None
  • Latest Release Status: Pre-Alpha
  • Project Type: Java Web Start application client
The "Domain Object Explorer" is a rich desktop application designed to ease the management of EJB3 entity beans. It auto-generates the user interface for the business objects on the fly (no need of additional GUI design/programming, althougth in can also be customized by developer), and it can be centrally managed from application server administration console (for datasource configuration, deployment of persistence units, and delivery of application/updates via Java Web Start technology).

Features
  • Pros:
    • Easy management and testing of EJB3 entity beans.
    • Rich user interface in zero-code (althought it can also be customized by developer)
    • No more HTML, JavaScript, nor AJAX handlers on server-side... only 100% pure Java on client side!!!
    • Auto-deployment of application/updates with Java Web Start technology.
    • Data sources can be centrally managed from application server administration console
    • ...
  • Cons:
    • Only runs on GlassFish v2 build 19 (M2)
    • Deployed aplications are really big (about 40Mb)
    • No stable version, yet
    • ...

Download
There is no archived release, yet. But you can checkout the "doe" projects from CVS:
  1. "doe4ejb3" is the core library (it also contains custom annotations for EJB3 entities).
  2. "doe4ejb3-app-client" is a Java EE 5 application client to be included in the users's enterprise application.
There is also a sample enterprise application (based on "trails" demos):
  1. "doe4ejb3-test-ejb3": demo EJB3 entity beans.
  2. "doe4ejb3-test-app" : enterprise application sample where "doe" is included to allow browsing/editing of the previous "EJB3" entities.

Installation guide
  1. Download, install and start GlassFish v2 application server (important note: "doe" has only been tested with GlassFish v2 build 19, prior versions may not work because of "issue 1173").
  2. Include the "doe4ejb3-app-client" into your enterprise application, and deploy the new EAR to the application server.
    Note: if you want to try with the sample application, you have to create a database for the EJB3 entities, and configure the connection parameters in GlassFish administration console (the JDBC datasource name should be "jdbc/doe4ejb3"), then open the projects in NetBeans 5.5 beta 2, and deploy the "doe4ejb3-test-app" project into the application server.
  3. Launch the new Java Web Start application client (note: you can use GlassFish administration console to browse the deployed application, and launch the new "doe4ejb3-app-client" module).
  4. See the instructions below for how to use the application.

Usage Instructions
You can launch the "doe" application client from GlassFish administration console (after installation, you also can run it from Java Web Start). The application has the next features:

Create a new entity object:

From "File" menu, select "New" option, and the class name of the new EJB3 entity. Then complete the entity properties and press the "Accept" button.

Search entity beans:

First, you have to select the persistence unit for the EJB3 entity class, and double click on the desired class to open the entity manager window on the desktop. Then select the query type ("All objects" or the name of a "custom named query" defined in the EJB3 entity bean). With custom named queries, you also have to complete the required parameters. Finally, press the "Search" button, and wait for the results (and the "done" indicator in the status bar).

Print the objects found:

After the search of the desired objects, click the "Print" button from the manager window, then configure page layout in printing dialog, and click the "Print" button from the printing dialog.

Browse/edit an object:

You have to select the object found, and click the "Edit" button from the manager window to view the entity details on the right panel (desktop). You can modify it's properties, and press the "Accept" button to save the changes.

Delete an object:

You have to select the object found, click the "Delete" button from the manager window, and confirm the deletion in the next window popup.

Roadmap
Future roadmap for the "doe" application:
  • English corrections (I don't speak English).
  • Improve "GUI" (menus, toolbars, better navigation, custom queries in search panel, copy&paste operations, drag&drop operations, ...).
  • Allow user extensions/plugins to add new actions/menus for entity objects.
  • Create custom editors for common property types (image viewers/loaders, dates, etc.)
  • Login configuration for database connections managed by client applications.
  • Printing support (entity lists and object details).
  • Better i18n support.
  • Load/unload of persistence units.
  • More sample applications.
